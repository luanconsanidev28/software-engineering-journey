Servidor fala com a api pois ele precisa saber se o usuário está no sistema, a api fala com o negócio onde ele vai fazer a autentcação do usuário e negócio depois fala com o banco para saber se o usuário realmente existe, depois da verificação o usuário ganha a autenticação e a autorização para mexer no sistema, daí o usuário quando colocar a foto para dentro do serviço o servidor irá chamar a API de novo, a api vai ver com o negócio se o usuário tem autenticação para fazer isso e com isso ele chama o banco depois de autenticado, para somente guardar o id, nome, tipo e data de upload depois, o negócio irá chamar um sistema onde ele tem um armazenamento em nuvem que guarda a foto do usuário, depois o negócio retorna para o servidor que deu certo e o servidor para o navegador, que mostra ao usuário uma tela de conclusão.
