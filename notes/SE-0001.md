Quais funcionalidades mínimas esse sistema precisa ter?
    Para um sistema baseado em no google drive teira que ter essas funcionalidades:
        1. cadastro e login;
        2. upload de arquivos de vários tipos;
        3. downloads desses arquivos para a máquina principal;
        4. link para compartilhar os arquivos entre outros usuários;
        5.criação de pastas.

Quantos usuários você imagina suportar inicialmente?
    Inicialmente uns 100 usuários.

Como os usuários vão se autenticar?
    Através do login com email + senha, ou SSO do google ou microsoft.

Onde os arquivos serão armazenados?
    Em um banco de dados postgres.

Como evitar que um usuário veja arquivos de outro?
    Com autenticação via tokens e alinha no banco de dados de uma chave estrangeira que guardaria o id do usuário ao qual aquele arquivo pertece.

Como você faria backup dos arquivos?
    Colocaria mais máquinas de banco em sistemas de clouds diferentes e semanalmente iria fazer um backup do sistema.

Como você monitoraria se o sistema está funcionando?
    Com os logs, e também usando ferramentas de observabilidade como openTelemetry.

Se o sistema crescer 100 vezes, o que provavelmente precisará mudar?
    Muito provávelmente o banco e também teria que ter um gerenciamento em fila pela quantidade de requisições de uploads que teriam.
